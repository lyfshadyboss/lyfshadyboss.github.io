Fedora 下使用 Swap 分区

23 May 2014

Swap 的作用

Swap 类似于 Windows 下的虚拟内存，本质上是一块磁盘空间或仅仅是一个文件，当内存不够时，系统将内存中的一部分优先 级较低的内容放到Swap 上，节省出一定的内存空间．这种特性在运行比较大的软件时很有用，避免了因内存不足而引起的程序 崩溃．

创建并使用 Swap 分区

如果在安装系统时没有创建 Swap 分区，可以手动创建它：

  1. 先划出一个分区，比如 /dev/sda9

  2. 格式化

    mkswap /dev/sda9

  3. 使用 swap

    swapon /dev/sda9

  4. 将 swap 写入 /etc/fstab，系统启动时自动开启

    查看 UUID
blkid /dev/sda9
=> /dev/sda9: UUID="6f814ec3-c10d-4aa5-aa8e-58bcd7e6a740" TYPE="swap" PARTUUID="120044c5-09"

编辑 /etc/fstab，写入 swap 纪录
echo UUID=6f814ec3-c10d-4aa5-aa8e-58bcd7e6a740      swap        swap        defaults    0   0

  5. 用 free 命令可以查看内存和 swap 的使用情况

    free
=>
             total       used       free     shared    buffers     cached
Mem:       3884884    3699856     185028     265520      71616    1587796
-/+ buffers/cache:    2040444    1844440
Swap:      8191996       4200    8187796
