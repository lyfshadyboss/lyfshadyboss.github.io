Java Annotation 是什么？

"A form of metadata, provide data about a program that is not part of the program itself."

oracle java documentation

Annotation 的作用

中文名一般称为 “注解”

  • 给编译器提供信息，如检查错误
  • 编译过程期处理，如代码生成
  • 运行时处理，动态的控制程序的行为

包、类、方法、变量、参数都可以被注解

注：JAVA 8 又引入了 type annotations。JAVA 8 以前，注解只能用在声明，JAVA 8 以后，任何使用类型的地方都可以使用注解

Annotation Basics

格式


@interface Annotation {
    String des() default "";
    int value() default 0;
}
  • 用 @interface 表示一个 annotation
  • 配置项名为方法名，且方法无参数，无方法体
  • 返回值为基本类型, String, Class, annotation, enum 或者它们的一维数组
  • 可以用 default 来声明默认值

不带参数，可以省略括号


@Entity
...
                

@Override
void mySuperMethod() { ... }

带参数，分别赋值


@SuppressWarnings(value = "unchecked")
void myMethod() { ... }

或者

@SuppressWarnings("unchecked")
void myMethod() { ... }
                

@Author(
    name = "Benjamin Franklin",
    date = "3/27/2003"
)
class MyClass() { ... }

内置的 annotations

  • @Deprecated
  • @Override
  • @SuppressWarnings
  • @SafeVarargs (JAVA 7)
  • @FunctionalInterface (JAVA 8)

元注解

修饰注解的注解称为元注解 (Meta Annotation)

  • @Retention
  • @Documented
  • @Target
  • @Inherited
  • @Repeatable

@Retention

表示注解的存储规则

  • RetentionPolicy.SOURCE
    存在 source code 级别，在 class 和 runtime 级别丢弃
  • RetentionPolicy.CLASS
    存在 source code 和 class 级别，在 runtime 级别丢弃
  • RetentionPolicy.RUNTIME
    存在 source code , class 和 runtime 级别

@Documented

是否被 javadoc 纪录，默认不纪录

@Target

指明注解修饰的类型

  • ElementType.ANNOTATION_TYPE
  • ElementType.CONSTRUCTOR
  • ElementType.FIELD
  • ElementType.LOCAL_VARIABLE
  • ElementType.METHOD
  • ElementType.PACKAGE
  • ElementType.PARAMETER
  • ElementType.TYPE

@Inherited

是否可以继承，默认为 false。意思是当一个类查询是否具有某个注解时，如果没有，可以从父类中寻找，仅修饰 class

@Repeatable

JAVA 8 引入的，是否可以重复使用


@Schedule(dayOfMonth="last")
@Schedule(dayOfWeek="Fri", hour="23")
public void doPeriodicCleanup() { ... }

处理 annotation

编译期处理

自定义注解处理器 (annotation processor) 处理编译期间的注解

自定义注解处理器


@SupportedAnnotationTypes({"com.lyf.annotation.PrintMe"})
public class MyProcessor extends AbstractProcessor {
    public boolean process(Set< ? extends TypeElement > annotations, RoundEnvironment env) {
        Messager messager = processingEnv.getMessager();
        for (TypeElement te : annotations) {
            for (Element e : env.getElementsAnnotatedWith(te)) {
                messager.printMessage(Diagnostic.Kind.NOTE, "Printing: " + e.toString());
            }
        }
        return true;
}

    @Override
    public SourceVersion getSupportedSourceVersion() {
        return SourceVersion.latestSupported();
    }
}
